The Reliable Per-Frame Pose Contract

Tickets #7 and #8 changed pose output from a display-oriented vector into a time-aware, model-independent measurement.

1. Why cached poses were removed

Reusing a previous pose can improve displayed FPS, but it corrupts motion analysis:

fresh pose     position changes
cached pose    position repeats → false zero velocity
next pose      position jumps   → false velocity spike

The new rule is strict: every successfully decoded frame begins with cleared pose state and runs a fresh inference. A failed result remains invalid.

Live input uses a monotonic clock. Recorded video uses media time with a frame/FPS fallback. `FrameTimeline` rejects duplicate or decreasing source times.

3. One canonical schema for every model

`PoseTypes.hpp` contains no OpenCV types. It defines:

  • 18 named `JointId` values;
  • floating-point 2D pixel positions;
  • confidence and validity;
  • optional depth and 3D positions;
  • person ID and bounding box;
  • frame dimensions, frame ID, and source time.

4. Model adapters protect downstream code

OpenPose channel 0  ─┐
OpenPose channel 1  ─┼→ OpenPoseCocoAdapter → JointId → Pose
OpenPose channel 2  ─┘

future ONNX output ───→ NewModelAdapter      → JointId → Pose

Tracking, squat analysis, and rendering depend on `JointId`, not raw channel numbers. Replacing the model therefore changes its adapter, not every consumer.
