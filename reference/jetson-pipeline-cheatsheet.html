Jetson Orin Nano Inference Optimization

Optimize the dominant cost first, preserve fresh measurements, then remove transfers.

StageCurrent pathPreferred optimizationWhen
Capture/decodeCPU-facing `VideoCapture`Jetson GStreamer hardware pathWhen compressed input is significant
ResizeCPU `cv::resize`Keep CPU initially; later CUDA resizeOnly when profiling justifies it
PreprocessCPU `blobFromImage`TensorRT/CUDA preprocessingAfter inference becomes fast
Pose inferenceCPU or CUDA FP16TensorRT lightweight ONNX modelHighest priority
PostprocessCPUKeep CPU initiallyUsually small
Render/outputCPUDisable for benchmarksOptimize only if measured

Recommended order

  1. Measure every stage with a fixed video.
  2. Enable OpenCV DNN CUDA FP16.
  3. Reduce model input size while preserving fresh inference on every frame.
  4. Replace OpenPose with a faster multi-person ONNX/TensorRT model.
  5. Add hardware decode, GPU preprocessing, or zero-copy only after profiling.
