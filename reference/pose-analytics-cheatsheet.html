Reliability requirements

  1. Use live capture time or recorded media time, not processing FPS.
  2. Require increasing frame IDs and fresh per-frame measurement state.
  3. Gate low-confidence joints and smooth trajectories before differentiating.
  4. Maintain a stable person ID in multi-person scenes.
  5. Use calibration, scale, or depth for metric speed and reliable 3D angles.

The current implementation never reuses an old pose: every successful input frame runs inference, and failed results remain explicitly invalid.
